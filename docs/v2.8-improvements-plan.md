# v2.8 Improvements — single-PR working plan

This document is the **source of truth** for the v2.8 round of fixes/features.
Everything below is delivered incrementally on **one branch / one PR**, one item
at a time. If work is interrupted, resume from the first unchecked box.

The five items come directly from user feedback:

1. **YTD is wrong** — the portfolio is *up* this year but the dashboard reports a
   negative YTD growth. Something is broken in the calculation.
2. **DAX ETF metadata is wrong** — the correct holding is the **Global X DAX
   Germany ETF**, ticker **`DAX`** (NASDAQ-listed), not `EXS1.DE` / iShares Core DAX.
3. **Pick instruments during onboarding, with validation** — let the user choose
   tickers during onboarding via a *validated* path so an invalid/incorrect
   symbol can't be seeded.
4. **Daily growth** — show the latest *completed trading day*'s growth in **both
   currencies**, fall back to the last business day (Friday's data on a Sunday;
   yesterday's data when opened pre-close in CET), **label the as-of date**, and
   handle the fact that some holdings are ETFs (daily close) and some are mutual
   funds (lagged NAV).
5. **Value-over-time line graph** — a classic portfolio-value line chart on the
   Overview page with **Day / Month / Year / All-time** range selectors.

---

## Root-cause notes (discovered during exploration)

- **Item 1 keystone bug:** `positions_service.compute_positions(as_of=…)` and
  `total_portfolio_value(as_of=…)` value holdings with
  `prices_service.latest_close()` — i.e. **today's** price — *regardless of the
  `as_of` date*. So the Jan-1 YTD start value, the MTD month-start value, and the
  entire analytics equity curve are all priced at today's close. When prices have
  risen YTD, the start value is overstated and YTD growth turns negative even
  though the user is up. Fixing the as-of price lookup fixes items 1, 4 and 5 at
  once (they all need historically-correct valuations).
- Default seed lives in `services/onboarding_service.py` (`DEFAULT_INSTRUMENTS`).
- FX already forward-fills via `domain/currency.lookup_rate_with_forward_fill`;
  prices need the equivalent as-of helper.
