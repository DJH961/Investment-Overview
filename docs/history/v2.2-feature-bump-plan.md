## Phase split

### (a) FX-aware historical valuations  ← **shipped in this PR**
Foundation. Without this, USD/DKK growth curves are wrong and risk metrics
in phase (c) compute on a falsified series.

Scope:
- Extend `SUPPORTED_CURRENCIES` to include `DKK`. Add the FX backfill for
  `EUR→DKK` to the boot flow (`boot.py` / wherever `refresh_fx_history` is
  invoked) and to `display_currency_service`.
- Generalise `fx_service.refresh_fx_history` to refresh a *list* of quotes
  per call (currently EUR↔USD hard-coded in callers).
- Add `domain/currency.py`:
  `convert_eur_to(amount_eur, target, fx_rate)` /
  `convert_native_to_eur(amount_native, native, fx_rate)` — pure helpers.
- Introduce a **historical FX-aware revaluation path** for the equity
  curve:
  - New service method `positions_service.total_portfolio_value(session,
    as_of, *, display_currency)` — currently only returns EUR; have it
    accept `display_currency` and convert each holding using *that day's*
    FX (not today's).
  - `snapshots_service` cache: extend the cache key to
    `(date, display_currency)` and store as columns `total_value_eur`,
    `total_value_usd`, `total_value_dkk`. Alembic migration adds new
    nullable columns and a `valued_in` discriminator (or simply add
    `total_value_usd`, `total_value_dkk` columns). Invalidate-on-write
    semantics stay the same.
- Update `_period_query.aggregate` and the monthly/yearly pages to pull
  the per-currency closing value from snapshots instead of multiplying
  the EUR series by today's FX.
- Update overview KPI: convert `total_value_eur` to display currency via
  the **as_of-day** rate, not `current_rate(quote="USD")`.
- "FX contribution to return" caption on the Analytics page placeholder.
- Tests: per-currency snapshot value uses date-of-snapshot FX, not spot.
  Regression: synthetic dataset where USD/EUR rate changes 30% over a
  year produces materially different USD-curve growth than EUR-curve.
- CHANGELOG entry under `[2.2.0] — Unreleased`.

Exit criteria for (a): yearly/monthly closing-value column flips when
display-currency toggle changes, and the change is **historical** not just
a global scalar.

### (b) Instrument auto-detect + manual transaction  ← **shipped in this PR**
Scope:
- Extend `ParsedTransactionRow` with optional `name`, `asset_class`,
  `expense_ratio`, `native_currency`.
- Vanguard XLSX parser: fill `name` (security description). Fidelity
  parser: fill `name`. Both: pass-through `asset_class` when knowable
  (sweep MMDA → cash, etc.).
- New `services/instrument_enrichment_service.py` that calls
  `yfinance_client` to fetch long name, quote type → asset_class,
  currency, and `annualReportExpenseRatio`. Cache by symbol; invoked
  lazily by `importer_service` for any field still missing after parse.
- `Instrument.asset_class` CheckConstraint: add `'unknown'`. Alembic
  migration.
- `InstrumentOverride`: add nullable `name_override`,
  `asset_class_override`, `expense_ratio_override`. Use in display.
- Transactions page: add **Add manual transaction** dialog + per-row
  edit that allows overriding the instrument's name/asset class/TER.
- Importer: stop hard-coding `asset_class="etf"`; use parse → enrichment
  → `'unknown'`.
- Tests covering: enrichment fallback path, override precedence in
  display, manual-entry round-trip, idempotent re-import preserves
  enriched fields.

### (c) Analytics page + growth-oriented charts  ← **shipped in this PR**

Implementation notes (post-ship):
- Risk-free rate uses **yfinance `^IRX`** (13-week US T-bill yield,
  percent-quoted) rather than Frankfurter; Frankfurter does not publish
  a risk-free yield series and the user explicitly rejected
  hypothetical fallbacks ("pulling from somewhere that doesn't have
  something isn't smart"). Cached for 24h in `app_config`; a manual
  override is supported and wins over the fetched value.
- Equity curve compares the portfolio against a rebased benchmark line
  (default `VT`, configurable in Settings) so the overlay reads as
  "what if I'd bought the index instead?" rather than as raw price
  units.
- Calmar uses the annualised mean of daily returns over the lookback
  window divided by `|max_drawdown|`; Ulcer is RMS of percentage
  drawdowns from running peak (not annualised — it's already
  scale-free).
- Per-instrument attribution rolls up start/end EUR values from
  snapshots, subtracts net BUY/SELL contributions in the window, and
  adds cash dividends. Sums to the portfolio's absolute P&L modulo
  intra-position rounding.
Scope:
- New page `/analytics` registered alongside the others; sidebar link.
- Compose existing `domain/risk.py` + `domain/returns.py` (TWR, vol,
  Sharpe, Sortino, max DD, win-rate, beta, alpha) — none of these are
  wired into UI today.
- Add new domain modules where needed:
  - `domain/risk_extras.py` — Calmar, Ulcer, historical VaR/CVaR,
    skew/kurtosis.
  - `domain/attribution.py` — per-instrument contribution to return.
- `services/benchmark_service.py` — fetches benchmark daily closes
  (default **VT**) via existing `yfinance_client`, caches via
  `price_cache_repo`. Symbol configurable in Settings (`app_config`
  key `benchmark_symbol`).
- `services/risk_free_service.py` — pulls ECB EST rate via Frankfurter
  (Frankfurter exposes EST under the same daily-rates endpoint when
  base is EUR and symbols include the rate code; if not, fall back to a
  3-month treasury proxy and surface a "fallback used" tag).
- Overview hero: equity curve with cumulative-contributions overlay,
  shaded gain area. Bigger Total Gain / Total Growth % cards. Add
  CAGR + TWR KPI cards.
- Monthly page: replace contributions bar with value+contribution
  combined chart; add per-period return %. Yearly page: same + add
  monthly-return heatmap.
- Settings page: benchmark symbol picker, FX refresh window (already
  partially present), display of risk-free rate source and current
  value.

## Conventions to remember
- ORM split across `LedgerBase` / `ConfigBase` / `CacheBase` — no
  cross-tier FKs. Cache-tier additions go in `models/` with `CacheBase`.
- Validation: `python -m pytest -q`, `ruff check .`, `ruff format --check .`,
  `mypy src/investment_dashboard/domain`. mypy strict only on `domain/`.
- Migrations live in `migrations/versions/`; one per logical change.
- Don't break the existing 2.1.4 EUR/USD pages — every new behaviour is
  additive behind the existing display-currency toggle (which now
  accepts DKK too).
