## Root causes (confirmed)

- **yfinance single-symbol bug** — `_download_window` reads `frame["Close"]` for
  a one-symbol fetch, but `group_by="ticker"` returns a `MultiIndex` even for a
  single ticker, so single-symbol fetches (benchmarks!) raise `KeyError` and are
  swallowed. This is why the Analytics benchmark and the Overview "Beating the
  market" VT number fail while bulk price fetches work. Fix: detect the
  `MultiIndex` and read `frame[symbol]["Close"]`.
- **FX per-trade-date** — already applied across Deposits / Transactions /
  metrics; re-verified. The silent `Decimal(1)` spot fallback in
  `positions_service` only affects EUR-conversion when *no* FX history exists at
  all (degrade-gracefully), not the per-trade-date paths.
- **Closing balance** — `snapshots_service.get_or_compute` values via
  `positions_service.total_portfolio_value(as_of=...)`, which now uses
  `compute_positions(as_of=...)` + `prices_service.close_as_of` and so already
  accounts for the share count *and* price in effect on each period end.
